### 7.2 流程实例主数据表 (BPM_MainData)

流程实例主数据，在流转中的流程实例存在本表中，流程结束后就会归档到 BPM_ArchivedData 表中去，这两张表必须保持一致性。

| 字段名称 | 主键 | 字段类型 | 长度 | 是否为空 | 默认值 | 字段说明 |
| :--- | :---: | :---: | :---: | :---: | :---: | :--- |
| WF_OrUnid | Y | varchar | 50 | N | 无 | 文档 32 位唯一标识一般与 WF_DocUNID 相等 |
| Subject | N | varchar | 300 | N | 无 | 文档标题，此字段发送待办邮件时必须 |
| WF_Appid | N | varchar | 30 | Y | 无 | 所属应用的 ID 号 |
| WF_Processid | N | varchar | 50 | Y | 无 | 此文档使用的流程 ID 号 |
| WF_ProcessName | N | varchar | 250 | Y | 无 | 流程名称 |
| WF_AddName | N | varchar | 30 | Y | 无 | 流程启动者的英文名 |
| WF_AddName_CN | N | varchar | 50 | Y | 无 | 流程启动者的中文名 |
| WF_Author | N | varchar | 8000 | Y | 无 | 当前审批者，此字段的内容会存入到 WF_AllAuthors 中 |
| WF_Author_CN | N | varchar | 8000 | Y | 无 | 当前审批者的中文名，与 WF_Author 一一对应 |
| WF_EndUser | N | varchar | 8000 | Y | 无 | 已审批结束的用户列表，此字段值会合并到 WF_AllReaders 中 |
| WF_AllReaders | N | varchar | 8000 | Y | 无 | 所有有权访问本文档的用户列表 |
| WF_CopyUser | N | varchar | 8000 | Y | 无 | 传阅用户列表 |
| WF_DocNumber | N | varchar | 30 | Y | 无 | 流水单号系统自动生成，当需要自定义流水号时可以覆盖此字段 |
| WF_CurrentNodeid | N | varchar | 100 | Y | 无 | 当前所在节点 id |
| WF_CurrentNodeName | N | varchar | 500 | Y | 无 | 流程当前所处的节点名称，当有多个时用逗号分隔，要想得到每个用户具体在那个环节请从 WF_CurUserStatus 字段中获取 |
| WF_Status | N | varchar | 30 | Y | 无 | 流程当前状态，选项：Current=审批中，End=已结束，Draft=草稿 |
| WF_VersionNumber | N | varchar | 30 | Y | ('') | 当前文档的版本号 |
| WF_SourceEntrustUserid | N | varchar | 200 | Y | 无 | 参与此文档的且设置了代办的用户，显示被代办文档时用 |
| WF_TargetEntrustUserid | N | varchar | 200 | Y | 无 | 代办的用户列表，记录那些用户是代办的 |
| WF_ProcessNumber | N | varchar | 30 | Y | 无 | 过程属性中指定的流程编号 |
| WF_MainNodeid | N | varchar | 50 | Y | ('') | 主流程中启动本节点 Nodeid 如果没有则为空值 |
| WF_MainDocUnid | N | varchar | 50 | Y | ('') | 如果是子流程则有主文档的唯一编号 |
| WF_TotalTime | N | varchar | 10 | Y | 无 | 总耗时 |
| WF_EndTime | N | varchar | 30 | Y | ('') | 文档办结时间，文档结束后会自动生成 |
| WF_BusinessNum | N | varchar | 50 | Y | ('') | 所属业务场景唯一标识 |
| WF_EndBusinessid | N | varchar | 30 | Y | ('') | 对应流程结束环节的业务状态标识 |
| WF_Folderid | N | varchar | 30 | Y | ('') | 归档时此字段从自动活动中得到指定的归档文件夹编号 |
| WF_TransFlag | N | varchar | 30 | Y | ('N') | 数据是否已传输到其他数据库表标记位 |
| WF_ArcFormNumber | N | varchar | 30 | Y | ('') | 归档后显示表单的编号 |
| WF_AddDeptid | N | varchar | 50 | Y | 无 | 起草人所在部门 ID 号 |
| WF_Systemid | N | varchar | 30 | Y | 无 | 启动本流程的业务系统的 ID 号 |
| WF_DocCreated | N | varchar | 30 | Y | 无 | 创建时间 |
| WF_LastModified | N | varchar | 50 | Y | ('') | 最后更新时间 |
| XmlData | N | xml | -1 | Y | 无 | 如果流程过程属性中选择 XmlData 表为数据主表，则系统会自动把表单字段组合成 xml 存入此字段中，否则会存入到 BPM_SubData 中 |

> **XmlData 字段内容说明：**
> 1. `unit` 为申请人单位
> 2. `Project` 为所属项目
> 3. `package` 为项目包
> 4. `Subproject` 为子项目
> 5. `ErrorType` 为故障类型或服务目录
> 6. `Appsystem` 为应用系统
> 7. `Detail` 为详情描述
